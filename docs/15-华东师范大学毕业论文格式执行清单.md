# 15 华东师范大学毕业论文格式执行清单

## 1. 格式要求来源

格式要求来自项目外课程资料，项目内保留转换后的可读版本：

```text
docs/format_requirements/华东师范大学毕业论文格式要求.docx
```

该文件已转换为可读版本：

```text
docs/format_requirements/华东师范大学毕业论文格式要求.docx
```

本清单根据转换后的文本逐条整理，用于指导最终 `paper.docx` 和 `paper.pdf` 的格式调整。

## 2. 论文构成要求

原要求指出论文应由封面、目录、正文三部分构成。正文部分包括：

1. 中外文题名
2. 中外文摘要
3. 中外文关键词
4. 正文
5. 参考文献
6. 致谢

如有附录，应装订在参考文献后面。

本文需要执行：

| 要求 | 当前处理 |
|---|---|
| 封面 | 保留课程论文封面字段，并嵌入原课程封面图 |
| 目录 | 需要在最终稿中增加目录 |
| 中文题名、摘要、关键词 | 需要保留并增强 |
| 英文题名、摘要、关键词 | 需要新增 |
| 正文 | 需要升级为 9000 字以上正文 |
| 参考文献 | 需要按引用顺序编号 |
| 致谢 | 需要新增 |
| 附录 | 需要放核心代码、附图附表路径和运行环境 |

## 3. 页面设置

| 项目 | 要求 |
|---|---|
| 纸型 | A4 |
| 方向 | 纵向，个别页面可横向 |
| 文档网格 | 无网格 |
| 上边距 | 2.5 cm |
| 下边距 | 2.0 cm |
| 左边距 | 3.0 cm |
| 右边距 | 2.5 cm |
| 装订线 | 左侧 |

最终 DOCX 应通过后处理脚本设置页面边距，而不是只依赖 Pandoc 默认值。

## 4. 段落与字体

| 项目 | 要求 |
|---|---|
| 正文中文字体 | 宋体 |
| 正文字号 | 小四 |
| 中文题名 | 黑体小三 |
| 外文题名 | 小三 |
| 各段标题 | 黑体小四，前后一致 |
| 外文字体 | Times New Roman |
| 行距 | 1.5 倍 |
| 段落缩进 | 首行缩进 2 字符 |
| 独立标题 | 不加标点 |

最终 DOCX 需要统一普通段落、标题、摘要、参考文献和附录代码的字体与行距。

## 5. 标题层次

理科类层次系统建议：

```text
第一层：1、2、3、……
第二层：1.1、2.1、3.1……
第三层：1.1.1、2.1.1、3.1.1……
第四层：1.1.1.1……
第五层：1）、2）、3）……
```

当前论文采用 Markdown 标题自动转 Word 标题。后续建议：

1. 正文标题尽量控制在三级以内。
2. 章节标题保持“第一章/第二章”也可接受课程论文语境，但若严格贴合格式，可在正文中使用数字化标题。
3. 标题不加句号、冒号等结尾标点。

## 6. 页码

要求每页插入阿拉伯数字页码，位于页面底端居中。

当前自动转换流程未必插入页码。最终需要通过 DOCX 后处理脚本添加页脚页码。

## 7. 图表格式

要求：

1. 表名放在表格正上方。
2. 图名放在图件正下方。
3. 图表名中外文对照。
4. 表格采用三线表形式。
5. 插图需在文中相应处直接给出。
6. 图件大小：半栏图小于 60mm，通栏图 100mm 至 130mm。

本文执行方式：

| 项目 | 执行策略 |
|---|---|
| 表题 | `表 X 中文标题 / Table X English title` 放表格上方 |
| 图题 | 图片后放 `图 X 中文标题 / Figure X English title` |
| 三线表 | Markdown 表转 DOCX 后需尽量统一边框；若自动转换不完美，至少保证表题、编号、内容完整 |
| 图像宽度 | 核心图统一控制在约 13 cm 以内 |
| 图表引用 | 正文必须先解释或引用，再放图表 |

## 8. 注释与引用

要求正文需要注释时可使用脚注或尾注，注码用方括号形式标出。本文主要采用参考文献顺序引用，不使用脚注。

正文引用规范：

1. 首次提到数据集时标注 `[1]`。
2. 提到原始论文时标注 `[2]`。
3. 提到 R、tidyverse、ggplot2、MASS、随机森林、LASSO 等方法时标注相应编号。
4. 参考文献按首次引用顺序排列。

## 9. 参考文献

格式要求：

| 类型 | 格式 |
|---|---|
| 专著 | `[序号] 作者.专著名[M].出版地：出版者，出版年，起止页.` |
| 期刊 | `[序号] 作者.题名[J].刊名.出版年，卷号(期号)：起止页.` |
| 电子文档 | `[序号] 作者.题名[E].出处或网址.发表或更新日期/引用日期.` |

本文至少需要 6 篇以上权威参考文献，并包含两篇以上外文文献。当前建议参考文献数为 11 篇，满足数量要求。

## 10. 附录要求

附录应包括论文形成过程中涉及的数据、图表、程序等材料。本文附录应包含：

1. R 运行环境。
2. 核心 R 分析代码。
3. 附图清单。
4. 附表清单。
5. 模型对象清单。

由于完整脚本较长，论文中放核心代码是必要要求；放不下的附图附表可在附录中说明路径。

## 11. 最终导出检查项

导出前必须检查：

1. DOCX 可打开，段落、表格、图片数量合理。
2. PDF 页数合理，文本可抽取。
3. 无明显空白页。
4. 无 `\newpage`、LaTeX 源码残留。
5. 表格编号连续。
6. 图片编号连续。
7. 表题在表上，图题在图下。
8. 参考文献编号与正文引用顺序一致。
9. 正文中文字符数满足 9000 字以上。
10. 封面信息仍需用户填入姓名、学号、学院、专业。

## 12. 与当前论文的差距

当前 `paper.docx` 已有较完整内容，但与严格格式仍存在差距：

1. 缺少英文摘要和英文关键词。
2. 缺少自动目录。
3. 图题当前主要作为图片 alt 文本，不完全符合“图名在图下”要求。
4. 表题缺少英文对照。
5. 页边距、行距、字体和页码需要通过 DOCX 后处理统一设置。
6. 参考文献需要按 `[序号]` 格式并在正文插入顺序引用。
7. 附录需要增加附图附表清单。

本轮升级应逐项修正这些差距。
