# 11 论文正文撰写原创性与参考文献方案

## 1. 本部分目标

本部分用于指导最终论文正文如何写，尤其关注正式论文格式、原创性、查重率和参考文献规范。课程要求明确说明最终提交到维普课程作业管理系统，重复率超过 20% 会判定为抄袭。因此，论文写作必须从一开始就围绕原创表达和真实分析结果组织。

目标：

1. 给出每一章正文写作重点。
2. 提供低重复率写作策略。
3. 规范引用数据来源和 R 工具。
4. 避免 AI 模板化表达。
5. 确保论文内容来自真实 R 分析结果。

## 2. 正文总体写作原则

1. **事实来自数据**：所有具体结论都要由 R 输出、图表或模型支撑。
2. **方法简洁说明**：不要大段解释 t 检验、回归、随机森林原理，课程重点是会用 R 分析数据。
3. **结果详细解释**：图表和模型输出要用自己的话解释。
4. **谨慎使用因果语言**：本文是观察性数据分析，不能直接得出严格因果结论。
5. **参考文献规范引用**：数据来源、R 软件和主要方法资料要列入参考文献。
6. **避免模板化开头**：少写“随着社会经济的快速发展”这类高重复率句式。

## 3. 各章节写作重点

### 摘要

摘要最后写，建议 300 至 500 字。

结构：

1. 数据来源和研究对象。
2. R 分析方法。
3. 主要发现。
4. 结论和应用意义。

示例骨架：

> 本文基于 UCI Machine Learning Repository 的 Bike Sharing Dataset，选取 2011 年至 2012 年美国华盛顿 D.C. 共享单车小时级租赁数据，使用 R 语言对共享单车租赁需求的影响因素和预测模型进行分析。本文首先进行数据预处理和描述性统计，然后通过可视化分析、统计检验、多元回归、计数模型和机器学习模型，考察时间因素、天气状况、环境变量和用户类型对租赁量的影响。研究发现，……。本文结果表明，R 语言能够较好支持从数据整理、统计建模到结果可视化的完整分析流程。

具体发现等真实结果出来后再填。

### 第一章 绪论

写作重点：

1. 共享单车是城市短途出行方式之一。
2. 租赁需求受时间、天气、工作日等因素影响。
3. 需求分析有助于车辆调度和运营管理。
4. 本文使用 R 进行综合数据分析。

避免：

- 不要写过长城市交通宏大背景。
- 不要复制共享单车行业报告。
- 不要写无法由数据支持的政策判断。

### 第二章 数据来源与变量说明

写作重点：

1. 数据来自 UCI。
2. 数据记录 2011 至 2012 年华盛顿 D.C. 共享单车租赁情况。
3. 主分析使用 `hour.csv`。
4. 说明样本量和变量数。
5. 用变量表解释变量。
6. 强调 `cnt = casual + registered`。

原创策略：

- 数据来源可以引用，但背景说明用自己的话。
- 变量表可以客观说明，不要复制 README 句子。

### 第三章 数据预处理与描述性统计

写作重点：

1. 读取数据并检查维度。
2. 检查缺失值和重复值。
3. 转换分类变量。
4. 构造派生变量。
5. 解释描述统计结果。

原创策略：

- 使用自己计算的数值。
- 结合表格解释，不写泛泛而谈。

### 第四章 探索性数据分析

写作重点：

1. 总租赁量分布。
2. 小时规律。
3. 工作日与非工作日模式。
4. 月份和季节差异。
5. 天气和环境变量。
6. 用户类型初步比较。

原创策略：

- 每张图都写自己的观察。
- 不要使用网上对 Bike Sharing Dataset 的常见结论原句。
- 图表解释多写“本文数据中表现为”。

### 第五章 统计推断分析

写作重点：

1. 对组间差异进行检验。
2. 说明为什么使用参数检验和非参数检验。
3. 报告 p 值和差异方向。
4. 说明大样本下显著性解释应谨慎。

原创策略：

- 不要复制统计检验定义。
- 重点解释本数据检验结果。

### 第六章 回归模型与影响因素分析

写作重点：

1. 说明因变量使用 `log1p(cnt)`。
2. 说明自变量选择。
3. 报告基准模型和交互项模型。
4. 解释主要变量。
5. 做模型诊断和稳健标准误。

原创策略：

- 用模型系数方向和显著性写具体结论。
- 不写教材式“线性回归是一种……”。

### 第七章 计数模型与预测模型比较

写作重点：

1. 说明 `cnt` 是计数变量。
2. 比较 Poisson 和负二项模型。
3. 说明过度离散。
4. 比较测试集预测指标。
5. 解释随机森林变量重要性。

原创策略：

- 预测指标来自自己运行结果，重复率低。
- 区分解释模型和预测模型。

### 第八章 用户类型差异分析

写作重点：

1. 注册用户与休闲用户规模差异。
2. 两类用户小时模式差异。
3. 工作日与非工作日下两类用户差异。
4. 两类用户模型比较。
5. 行为机制解释。

原创策略：

- 这是最适合写出个人分析的章节。
- 多结合图表而不是泛泛说“用户需求不同”。

### 第九章 结论与建议

写作重点：

1. 按研究问题总结结论。
2. 给共享单车运营建议。
3. 说明研究不足。
4. 提出未来扩展方向。

建议结论分条：

1. 时间因素结论。
2. 天气环境结论。
3. 工作日差异结论。
4. 用户类型结论。
5. 模型比较结论。

不足：

1. 数据只来自一个城市。
2. 缺少站点位置和车辆投放信息。
3. 缺少价格、活动、节假日具体事件等变量。
4. 本文主要分析相关关系，不做严格因果识别。

## 4. 查重率控制策略

### 4.1 高风险内容

以下内容容易重复：

1. 共享单车背景介绍。
2. UCI 数据集说明。
3. 统计方法定义。
4. R 包功能介绍。
5. 通用结论句式。

### 4.2 低风险内容

以下内容重复率较低：

1. 自己计算的描述统计结果。
2. 自己绘制图表后的解释。
3. 自己的模型结果和指标。
4. 对 `casual` 和 `registered` 的具体比较。
5. 针对本数据提出的运营建议。

### 4.3 写作技巧

1. 背景控制在全文 10% 以内。
2. 方法原理控制在全文 15% 以内。
3. 结果分析占全文 50% 以上。
4. 讨论和建议占全文 15% 至 20%。
5. 每个结论都引用自己的图号、表号或模型。

## 5. 禁用和慎用表达

慎用：

- “随着社会经济的快速发展”
- “大数据时代背景下”
- “具有十分重要的理论和现实意义”
- “本文采用先进的方法”
- “结果完全证明”
- “显著影响因素导致”

推荐：

- “本文数据中显示”
- “从图 X 可以看出”
- “在控制其他变量后”
- “该变量与租赁量呈正相关”
- “结果提示”
- “可能反映”
- “需要结合更多数据进一步验证”

## 6. 参考文献建议

至少列以下参考文献：

### 数据集

```text

```

### 原始论文

```text
Fanaee-T, H., & Gama, J. (2013). Event labeling combining ensemble detectors and background knowledge. Progress in Artificial Intelligence. https://doi.org/10.1007/s13748-013-0040-3
```

### R 软件

```text
R Core Team. R: A language and environment for statistical computing. R Foundation for Statistical Computing. https://www.r-project.org/
```

### tidyverse

```text
Wickham, H., et al. Welcome to the tidyverse. Journal of Open Source Software, 2019.
```

### ggplot2

```text
Wickham, H. ggplot2: Elegant Graphics for Data Analysis. Springer.
```

### tidymodels

```text
Kuhn, M., & Wickham, H. Tidymodels: a collection of packages for modeling and machine learning using tidyverse principles. https://www.tidymodels.org/
```

参考文献数量建议 6 至 10 篇，不要太少，也不要堆太多无关文献。

## 7. 正文引用方式

数据来源引用：

> 本文数据来自 UCI Machine Learning Repository 的 Bike Sharing Dataset。

方法工具引用：

> 数据整理和可视化主要基于 R 语言及 tidyverse 相关工具包完成。

不要每个 R 包都在正文中反复引用，可在方法部分统一说明，参考文献中列出主要工具即可。

## 8. 图表编号规则

建议：

```text
表 1 数据集基本信息
表 2 变量说明
表 3 主要变量描述性统计
表 4 统计检验结果
表 5 线性回归模型结果
表 6 计数模型比较
表 7 预测模型性能比较
表 8 用户类型模型比较
```

图：

```text
图 1 总租赁量分布
图 2 不同小时平均租赁量
图 3 工作日与非工作日小时模式
图 4 月份平均租赁量
图 5 不同季节租赁量分布
图 6 天气状况与租赁量
图 7 环境变量与租赁量关系
图 8 相关系数热力图
图 9 预测值与真实值对比
图 10 用户类型小时模式
```

图表不要过多，正文中 8 至 12 张图表比较合适。其余放附录或不放。

## 9. 附录代码处理

附录代码要：

1. 删除临时调试代码。
2. 保留包加载、路径设置、数据读取、分析和输出。
3. 保留必要注释。
4. 不贴过长无用输出。
5. 保留 `sessionInfo()`。

附录标题：

```text
附录 A R 分析代码
附录 B R 运行环境信息
```

## 10. 最终写作顺序

不要从摘要开始写。推荐顺序：

1. 先跑 R 脚本，生成所有图表和表格。
2. 写第二章数据来源和变量说明。
3. 写第三章数据预处理和描述统计。
4. 写第四章探索性分析。
5. 写第五章统计推断。
6. 写第六、七、八章模型结果。
7. 写第九章结论。
8. 最后写摘要、关键词和引言。
9. 整理参考文献和附录。
