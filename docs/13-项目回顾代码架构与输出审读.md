## 3. `analysis.R` 代码流程审读

`scripts/analysis.R` 是项目核心分析脚本。脚本按模块顺序组织：

1. 环境准备：设置随机种子，检查 R 包可用性，定义输出目录。
2. 数据读取：读取 `hour.csv` 和 `day.csv`。
3. 数据质量检查：输出数据集规模、缺失值、重复值、`cnt = casual + registered` 业务一致性检查。
4. 变量转换：将编码变量转换为中文因子标签，还原温度、湿度、风速实际量纲，构造 `cnt_log`、用户占比和小时段。
5. 描述统计：输出连续变量统计、分类变量频数、分组均值、小时和月份汇总。
6. 可视化：生成分布图、小时趋势图、工作日对比图、月度图、季节箱线图、天气箱线图、环境变量平滑图、相关系数热力图、用户类型图、日级趋势图。
7. 统计推断：执行 t 检验、Wilcoxon 检验、ANOVA、Kruskal-Wallis 检验、Tukey 事后比较。
8. 回归模型：建立 `log1p(cnt)` 线性回归、工作日与小时交互模型，输出模型拟合、重点系数、VIF、Breusch-Pagan 异方差检验、HC3 稳健标准误和残差诊断图。
9. 计数模型：建立 Poisson 回归和负二项回归，计算 Poisson 过度离散比，输出 AIC/BIC 和负二项重点变量发生率比。
10. 预测模型：划分训练集和测试集，比较线性回归、负二项回归、LASSO、随机森林和回归树，输出 RMSE、MAE、R 平方、预测散点图和变量重要性。
11. 用户类型分析：分别分析 `casual` 与 `registered`，建立用户类型模型并输出重点系数。
12. 结果摘要：输出 `table99_result_summary.csv` 和 `sessionInfo.txt`。

## 4. 关键专业边界

### 4.1 数据泄露边界

`cnt = casual + registered` 是本项目最重要的数据建模边界。`casual` 与 `registered` 是总租赁量的组成部分，因此不能作为预测 `cnt` 的解释变量。当前脚本在 `cnt` 模型中没有加入二者，只在用户差异章节将其分别作为因变量使用，处理方式正确。

### 4.2 主模型变量设定

当前主回归模型使用：

```r
cnt_log ~ season + yr + mnth + hr + holiday + workingday + weathersit +
  temp_celsius + hum_percent + windspeed_real
```

没有同时纳入 `weekday`、`holiday`、`workingday` 三者，是因为 `workingday` 与周末、节假日存在定义关系。`weekday` 主要用于热力图和探索性分析，主模型保留 `holiday` 与 `workingday`，避免设计矩阵秩亏。

### 4.3 解释模型与预测模型分工

线性回归和负二项回归主要服务于解释：它们提供系数方向、显著性、发生率比和模型诊断。随机森林、LASSO 和回归树主要服务于预测：它们通过测试集 RMSE、MAE、R 平方评价泛化性能。论文中应明确区分两类模型，不能用预测性能直接替代统计解释。

核心结果包括：

1. 主数据 `hour.csv` 包含 17,379 条记录和 17 个原始变量。
2. 数据无缺失值、无重复记录，`cnt = casual + registered` 全部成立。
3. `cnt` 均值为 189.463，中位数为 142，最大值为 977，分布右偏。
