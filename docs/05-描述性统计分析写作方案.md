# 05 描述性统计分析写作方案

## 1. 本部分目标

描述性统计是论文中“对数据集描述的清楚程度”的核心支撑。它的作用不是简单罗列均值，而是让读者在进入模型前先理解数据的规模、分布、集中趋势、离散程度和变量之间的基本关系。

本部分应解决以下问题：

1. 总租赁量 `cnt` 的典型水平是多少？
2. 租赁量分布是否对称，是否存在右偏？
3. 休闲用户和注册用户在总租赁量中分别占什么位置？
4. 温度、湿度、风速等环境变量的取值范围如何？
5. 哪些变量后续适合做分组比较，哪些适合做连续变量建模？

## 2. 应分析的变量

### 2.1 主要连续变量

| 变量 | 含义 | 描述重点 |
| --- | --- | --- |
| `cnt` | 总租赁量 | 本文主要因变量，重点描述 |
| `casual` | 休闲用户租赁量 | 用于用户差异分析 |
| `registered` | 注册用户租赁量 | 用于用户差异分析 |
| `temp_celsius` | 气温 | 与租赁量的环境关系 |
| `atemp_celsius` | 体感温度 | 与气温比较，注意共线性 |
| `hum_percent` | 湿度 | 可能负向影响租赁量 |
| `windspeed_real` | 风速 | 可能负向影响租赁量 |

### 2.2 主要分类变量

| 变量 | 含义 | 描述重点 |
| --- | --- | --- |
| `season` | 季节 | 各季节样本是否均衡 |
| `yr` | 年份 | 2011 与 2012 样本量 |
| `mnth` | 月份 | 月度分布 |
| `hr` | 小时 | 各小时记录数是否接近 |
| `workingday` | 是否工作日 | 工作日与非工作日占比 |
| `holiday` | 是否节假日 | 节假日样本较少 |
| `weathersit` | 天气状况 | 恶劣天气样本极少 |

## 3. 已知初步统计事实

根据本地数据初步检查，`hour.csv` 的主要变量统计如下：

| 变量 | 最小值 | 第一四分位数 | 中位数 | 均值 | 第三四分位数 | 最大值 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| `cnt` | 1 | 40 | 142 | 189.46 | 281 | 977 |
| `casual` | 0 | 4 | 17 | 35.68 | 48 | 367 |
| `registered` | 0 | 34 | 115 | 153.79 | 220 | 886 |
| `temp` | 0.02 | 0.34 | 0.50 | 0.50 | 0.66 | 1.00 |
| `atemp` | 0.00 | 0.3333 | 0.4848 | 0.4758 | 0.6212 | 1.00 |
| `hum` | 0.00 | 0.48 | 0.63 | 0.6272 | 0.78 | 1.00 |
| `windspeed` | 0.00 | 0.1045 | 0.1940 | 0.1901 | 0.2537 | 0.8507 |

可直接转化为论文观察：

1. `cnt` 的均值 189.46 高于中位数 142，说明总租赁量分布存在右偏。
2. 注册用户的平均租赁量明显高于休闲用户，说明注册用户是共享单车系统的主要需求来源。
3. 租赁量最大值达到 977，说明高峰时段需求远高于一般水平。
4. 环境变量覆盖范围较广，适合用于分析天气和气候因素对租赁需求的影响。

## 4. R 描述统计代码

### 4.1 使用 dplyr 生成描述统计表

```r
desc_vars <- bike_hour |>
  select(cnt, casual, registered, temp_celsius, atemp_celsius, hum_percent, windspeed_real)

desc_table <- desc_vars |>
  summarise(
    across(
      everything(),
      list(
        n = ~ sum(!is.na(.x)),
        mean = ~ mean(.x, na.rm = TRUE),
        sd = ~ sd(.x, na.rm = TRUE),
        min = ~ min(.x, na.rm = TRUE),
        q1 = ~ quantile(.x, 0.25, na.rm = TRUE),
        median = ~ median(.x, na.rm = TRUE),
        q3 = ~ quantile(.x, 0.75, na.rm = TRUE),
        max = ~ max(.x, na.rm = TRUE)
      ),
      .names = "{.col}_{.fn}"
    )
  ) |>
  pivot_longer(
    cols = everything(),
    names_to = c("variable", ".value"),
    names_pattern = "(.+)_(n|mean|sd|min|q1|median|q3|max)"
  )

write_csv(desc_table, file.path(table_dir, "table04_descriptive_stats.csv"))
```

## 5. 必做图表

### 图 1：总租赁量分布图

目的：

- 展示 `cnt` 的右偏分布。
- 为后续使用 `log1p(cnt)` 提供理由。

代码：

```r
p_cnt_dist <- ggplot(bike_hour, aes(x = cnt)) +
  geom_histogram(bins = 50, fill = "#3B82F6", color = "white") +
  geom_density(aes(y = after_stat(count)), color = "#DC2626", linewidth = 0.8) +
  labs(
    title = "总租赁量分布",
    x = "每小时总租赁量",
    y = "频数"
  ) +
  theme_minimal()

ggsave(file.path(fig_dir, "fig01_cnt_distribution.png"), p_cnt_dist, width = 8, height = 5, dpi = 300)
```

论文解释示例：

> 从总租赁量分布可以看出，多数小时的租赁量集中在较低至中等水平，但右侧存在较长尾部，说明少数高峰时段的租赁量显著高于一般时段。这与描述性统计中均值高于中位数的结果一致。基于这一分布特征，本文在后续线性回归中使用 `log1p(cnt)` 作为因变量，以降低右偏分布对模型估计的影响。

### 图 2：用户类型分布比较

目的：

- 比较 `casual`、`registered`、`cnt`。
- 说明注册用户是主要需求来源。

代码：

```r
user_long <- bike_hour |>
  select(casual, registered, cnt) |>
  pivot_longer(
    cols = everything(),
    names_to = "user_type",
    values_to = "rentals"
  ) |>
  mutate(
    user_type = recode(
      user_type,
      casual = "休闲用户",
      registered = "注册用户",
      cnt = "总租赁量"
    )
  )

p_user_dist <- ggplot(user_long, aes(x = rentals, fill = user_type)) +
  geom_histogram(bins = 50, alpha = 0.7, position = "identity") +
  facet_wrap(~ user_type, scales = "free_y") +
  labs(
    title = "不同用户类型租赁量分布",
    x = "每小时租赁量",
    y = "频数"
  ) +
  theme_minimal() +
  theme(legend.position = "none")
```

### 6.1 年份

已知：

- 2011 年：8,645 条
- 2012 年：8,734 条

解释：

> 两个年份的观测数量接近，说明数据在年度维度上较为均衡，适合比较 2011 年和 2012 年租赁需求差异。

### 6.2 工作日

已知：

- 非工作日：5,514 条
- 工作日：11,865 条

解释：

> 工作日观测数量多于非工作日，这与一周中工作日天数较多相一致。后续分析将重点比较工作日和非工作日的小时租赁模式。

### 6.3 天气状况

已知：

- 晴朗或少云：11,413 条
- 多云或有雾：4,544 条
- 小雨雪：1,419 条
- 恶劣天气：3 条

解释：

> 天气状况以晴朗或少云为主，恶劣天气记录极少。因此，本文在解释恶劣天气对应结果时保持谨慎，避免根据极少数样本得出过强结论。

## 8. 描述性统计章节写作框架

建议第三章中描述统计部分按以下顺序写：

1. 先说明样本量和变量数。
2. 再说明缺失值和数据完整性。
3. 展示连续变量描述统计表。
4. 解释 `cnt` 的分布特征。
5. 对比 `casual` 和 `registered`。
6. 简要说明环境变量。
7. 引出后续探索性分析。

示例段落：

> 从描述性统计结果看，小时级总租赁量 `cnt` 的均值为 189.46，中位数为 142，最大值为 977。均值明显高于中位数，说明租赁量分布存在一定右偏，即大多数小时租赁量处于中低水平，但部分高峰时段租赁量显著较高。休闲用户平均租赁量为 35.68，注册用户平均租赁量为 153.79，说明注册用户构成共享单车系统的主要使用群体。

## 9. 常见错误

1. **只放表不解释**。表格必须配文字解释。
2. **解释过度**。描述统计只能说明分布特征，不能直接证明因果关系。
3. **忽略右偏分布**。这是后续对数回归的重要理由。
4. **把标准化变量当实际单位解释**。必须先还原再解释。
5. **忽略恶劣天气样本数过少**。这个点体现分析谨慎性。
