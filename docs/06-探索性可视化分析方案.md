# 06 探索性可视化分析方案

## 1. 本部分目标

探索性可视化分析是展示 R 语言优势的关键部分。R 的 `ggplot2` 非常适合将时间、天气、季节和用户类型等因素转化为直观图形。本部分不仅要生成图，还要让每张图对应一个可写进论文的发现。

本部分要回答：

1. 共享单车租赁量在一天 24 小时内如何变化？
2. 工作日和非工作日的小时模式是否不同？
3. 租赁量在月份和季节上是否有周期性？
4. 天气状况、温度、湿度、风速与租赁量有什么关系？
5. 休闲用户和注册用户的可视化模式是否不同？

## 3. 图形风格统一规范

建议统一主题：

```r
theme_paper <- function() {
  theme_minimal(base_size = 12) +
    theme(
      plot.title = element_text(face = "bold", hjust = 0.5),
      plot.subtitle = element_text(hjust = 0.5),
      legend.position = "bottom",
      panel.grid.minor = element_blank()
    )
}
```

颜色原则：

1. 不要使用过多颜色。
2. 分组变量使用稳定配色。
3. 工作日与非工作日使用对比明显但不刺眼的颜色。
4. 图表标题要清楚，不使用夸张描述。

## 4. 时间规律图

### 4.1 按小时平均租赁量

目的：

- 展示早晚高峰。
- 说明小时变量是核心解释变量。

代码：

```r
hour_summary <- bike_hour |>
  group_by(hour_num) |>
  summarise(
    mean_cnt = mean(cnt),
    median_cnt = median(cnt),
    .groups = "drop"
  )

p_hourly <- ggplot(hour_summary, aes(x = hour_num, y = mean_cnt)) +
  geom_line(color = "#2563EB", linewidth = 1) +
  geom_point(color = "#2563EB", size = 2) +
  scale_x_continuous(breaks = 0:23) +
  labs(
    title = "不同小时的平均共享单车租赁量",
    x = "小时",
    y = "平均租赁量"
  ) +
  theme_paper()
```

已知初步事实：

- 4 时平均租赁量最低，约 6.4。
- 8 时平均租赁量约 359。
- 17 时平均租赁量最高，约 461.5。
- 18 时也较高，约 425.5。

论文解释：

> 按小时统计的平均租赁量呈现明显的日内周期。凌晨时段租赁需求最低，早上 7 至 9 时快速上升，下午 17 至 18 时达到全天高峰。这一结果说明共享单车使用具有明显通勤属性，小时变量应作为后续建模的重要解释变量。

### 4.2 工作日与非工作日小时模式

目的：

- 说明工作日早晚高峰更明显。
- 说明非工作日可能中午至下午更平缓。

代码：

```r
working_hour_summary <- bike_hour |>
  group_by(workingday, hour_num) |>
  summarise(mean_cnt = mean(cnt), .groups = "drop")

p_working_hour <- ggplot(
  working_hour_summary,
  aes(x = hour_num, y = mean_cnt, color = workingday)
) +
  geom_line(linewidth = 1) +
  geom_point(size = 1.8) +
  scale_x_continuous(breaks = 0:23) +
  labs(
    title = "工作日与非工作日的小时租赁模式比较",
    x = "小时",
    y = "平均租赁量",
    color = "日期类型"
  ) +
  theme_paper()
```

论文解释重点：

1. 工作日曲线通常具有更尖锐的早晚高峰。
2. 非工作日曲线可能在白天较平缓，更接近休闲出行。
3. 该图可支持后续加入 `workingday:hr` 交互项。

### 5.2 季节箱线图

```r
p_season <- ggplot(bike_hour, aes(x = season, y = cnt, fill = season)) +
  geom_boxplot(alpha = 0.8, outlier.alpha = 0.25) +
  labs(
    title = "不同季节的租赁量分布",
    x = "季节",
    y = "每小时总租赁量"
  ) +
  theme_paper() +
  theme(legend.position = "none")
```

已知均值：

- 春季约 111.11
- 夏季约 208.34
- 秋季约 236.02
- 冬季约 198.87

论文解释：

> 从季节分组结果看，春季平均租赁量较低，夏季和秋季租赁量明显较高，说明季节因素与共享单车需求存在较强关联。季节差异可能与气温、降水、日照和居民出行意愿有关。

注意：这里仍然是相关解释，不是严格因果。

## 6. 星期与小时热力图

目的：

- 展示二维时间结构。
- 比单独折线图更能体现 R 可视化优势。

代码：

```r
weekday_hour <- bike_hour |>
  group_by(weekday, hour_num) |>
  summarise(mean_cnt = mean(cnt), .groups = "drop")

p_heatmap <- ggplot(weekday_hour, aes(x = hour_num, y = weekday, fill = mean_cnt)) +
  geom_tile(color = "white") +
  scale_x_continuous(breaks = 0:23) +
  scale_fill_viridis_c(option = "C") +
  labs(
    title = "星期与小时维度下的平均租赁量热力图",
    x = "小时",
    y = "星期",
    fill = "平均租赁量"
  ) +
  theme_paper()
```

论文解释：

> 热力图进一步显示，租赁需求不仅存在小时差异，也存在星期差异。工作日的早晚高峰色块更集中，而周末的高需求时段可能更分散，说明共享单车在通勤和休闲两种场景下具有不同使用模式。

## 8. 相关系数热力图

建议选择连续变量：

```r
corr_data <- bike_hour |>
  select(cnt, casual, registered, temp_celsius, atemp_celsius, hum_percent, windspeed_real)

corr_matrix <- cor(corr_data, use = "complete.obs")

png(file.path(fig_dir, "fig12_correlation_heatmap.png"), width = 2400, height = 1800, res = 300)
corrplot::corrplot(
  corr_matrix,
  method = "color",
  type = "upper",
  addCoef.col = "black",
  tl.col = "black",
  tl.srt = 45
)
dev.off()
```

解释重点：

1. `cnt` 与 `registered` 高度相关，但这是因为 `registered` 是 `cnt` 的组成部分，不用于预测 `cnt`。
2. `temp_celsius` 与 `atemp_celsius` 可能高度相关，建模时不宜同时作为核心解释变量，避免共线性。
3. 湿度、风速与租赁量可能存在负相关或弱相关，需要进一步通过回归控制其他因素。

## 9. 用户类型可视化

### 9.1 休闲用户与注册用户小时曲线

```r
user_hour <- bike_hour |>
  group_by(hour_num) |>
  summarise(
    casual = mean(casual),
    registered = mean(registered),
    .groups = "drop"
  ) |>
  pivot_longer(
    cols = c(casual, registered),
    names_to = "user_type",
    values_to = "mean_rentals"
  ) |>
  mutate(user_type = recode(user_type, casual = "休闲用户", registered = "注册用户"))

p_user_hour <- ggplot(user_hour, aes(x = hour_num, y = mean_rentals, color = user_type)) +
  geom_line(linewidth = 1) +
  geom_point(size = 1.8) +
  scale_x_continuous(breaks = 0:23) +
  labs(
    title = "休闲用户与注册用户的小时租赁模式",
    x = "小时",
    y = "平均租赁量",
    color = "用户类型"
  ) +
  theme_paper()
```

解释重点：

- 注册用户更可能体现通勤高峰。
- 休闲用户可能在白天或周末更活跃。

## 10. 可视化章节写作方式

每张图后建议按三句话写：

1. **图显示了什么现象**。
2. **这个现象可能意味着什么**。
3. **它如何连接到后续检验或建模**。

示例：

> 图 3 显示，共享单车租赁量在一天内具有明显周期性，凌晨时段需求最低，早晚通勤时段需求明显上升。该现象说明共享单车不仅具有休闲属性，也承担一定通勤功能。因此，后续回归模型中将小时变量作为重要分类解释变量，并进一步考察工作日与小时之间的交互作用。

## 11. 原创性与查重提醒

可视化解释最容易原创，因为它来自自己的图表结果。写作时应：

1. 使用“本文计算发现”“从图中可以看出”等结果导向表达。
2. 避免复制网络上关于共享单车的一般性论述。
3. 不写过长背景解释。
4. 每张图都结合具体峰值、趋势、差异或分布特征。
5. 对不确定原因使用“可能”“倾向于”“提示”，避免绝对化。
