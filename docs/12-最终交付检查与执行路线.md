# 12 最终交付检查与执行路线

## 1. 本部分目标

本部分是最终执行清单，用于确保后续按照 12 个文档可以完整完成课程论文。它把数据、R 代码、图表、正文、Word、PDF 和提交要求串起来。

最终目标：

1. 生成完整 R 分析脚本。
2. 生成论文所需全部图表和表格。
3. 写出正式课程论文。
4. 将论文转为 Word 可编辑稿。
5. 将 Word 转为 PDF 提交稿。
6. 检查格式、页数、图表、附录和引用。
7. 尽可能降低查重风险。

## 4. 具体执行路线

### 第一步：建立目录

```bash
mkdir -p scripts outputs/figures outputs/tables outputs/models paper
```

### 第二步：确认 R 可运行

在 RStudio 或终端中运行：

```r
R.version.string
```

如果使用终端：

```bash
Rscript --version
```

当前终端曾出现 `Rscript: command not found`，因此后续需要确认 R 安装路径或直接使用 RStudio。

### 第三步：安装 R 包

在 R 中运行：

```r
packages <- c(
  "tidyverse", "lubridate", "janitor", "skimr", "psych",
  "GGally", "corrplot", "patchwork", "scales", "rstatix",
  "broom", "modelsummary", "gt", "car", "lmtest",
  "sandwich", "MASS", "tidymodels", "ranger", "vip"
)

install.packages(setdiff(packages, rownames(installed.packages())))
```

不要使用 `sudo` 安装 R 包。

### 第四步：编写 `scripts/analysis.R`

脚本模块：

```r
# 0. 加载包和路径
# 1. 读取数据
# 2. 数据质量检查
# 3. 变量转换
# 4. 描述统计
# 5. 可视化
# 6. 统计检验
# 7. 线性回归
# 8. 计数模型
# 9. 预测模型
# 10. 用户类型分析
# 11. 保存 sessionInfo()
```

### 第五步：运行脚本并检查输出

检查：

```text
outputs/figures/
outputs/tables/
outputs/sessionInfo.txt
```

必须确认：

- 图片能正常打开。
- 表格有内容。
- 模型没有明显报错。
- 中文图名不乱码。
- 没有把 `casual` 和 `registered` 用作 `cnt` 预测变量。

### 第六步：写论文正文

先写结果章节，再写摘要和引言。

推荐顺序：

1. 第二章 数据来源与变量说明
2. 第三章 数据预处理与描述统计
3. 第四章 探索性数据分析
4. 第五章 统计推断分析
5. 第六章 回归模型
6. 第七章 计数模型与预测模型
7. 第八章 用户类型差异
8. 第九章 结论
9. 第一章 绪论
10. 摘要和关键词

## 5. Word/PDF 生成路线

根据本机约定，正式中文文档推荐：

```text
Markdown 源稿 -> DOCX 可编辑稿 -> PDF 交付稿
```

命令：

```bash
python scripts/build_final_docx.py
codex-docx-to-pdf paper/paper.docx paper
```

如果论文中有数学公式，应确保最终 DOCX/PDF 中没有 LaTeX 源码残留。本论文公式不多，主要可能出现：

```text
cnt = casual + registered
log1p(cnt)
RMSE
MAE
R 平方
```

这些公式可以直接用普通文本或 Word 公式表达，避免复杂 LaTeX。

## 6. 文档检查命令

生成 DOCX 后检查：

```bash
codex-docx-inspect paper/paper.docx
```

生成 PDF 后检查：

```bash
codex-pdf-inspect paper/paper.pdf
```

检查重点：

1. DOCX 段落和表格数量合理。
2. PDF 页数正常。
3. PDF 文本可抽取。
4. 没有明显空白页。
5. 图片没有跨页异常。
6. 中文字体正常。
7. 附录代码没有乱码。

## 7. 最终论文质量标准

### 7.1 内容完整

必须包含：

- 数据来源
- 变量说明
- 数据预处理
- 描述统计
- 可视化分析
- 统计检验
- 回归模型
- 计数模型或预测模型
- 用户类型差异
- 结论与不足
- 参考文献
- R 代码附录

### 7.2 图表完整

建议正文包含：

- 6 至 10 张图
- 6 至 8 张表

不要过度堆图。每张图表都必须被正文解释。

### 7.3 代码完整

附录代码必须：

- 可读
- 可复现
- 没有调试垃圾
- 有必要注释
- 包含路径设置
- 包含 `sessionInfo()`

## 8. 查重提交前自查

提交前逐项检查：

1. 背景部分是否过长。
2. 是否有大段复制 UCI README。
3. 是否有大段复制 R 包文档或统计教材。
4. 图表解释是否是自己的观察。
5. 模型结果是否结合具体输出。
6. 结论是否与前文结果对应。
7. 参考文献是否规范列出。
8. 是否有 AI 味很重的模板句。

建议把背景和方法原理压缩，把篇幅留给真实结果。

## 11. 风险与应对

| 风险 | 应对 |
| --- | --- |
| R 包安装失败 | 优先保留 tidyverse、ggplot2、MASS、broom 等核心包 |
| 随机森林调参耗时 | 使用固定参数随机森林 |
| 中文图表乱码 | 改用英文图中文字，论文正文中文解释 |
| 模型结果过长 | 正文只放核心变量，完整结果放附录 |
| 查重率偏高 | 压缩背景，重写方法说明，增加真实结果解释 |
| PDF 图表错位 | 先生成 DOCX，人工检查后再转 PDF |
