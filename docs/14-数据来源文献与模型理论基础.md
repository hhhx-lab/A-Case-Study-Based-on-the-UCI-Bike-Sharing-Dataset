# 14 数据来源、文献依据与模型理论基础

## 1. 数据来源与原始研究背景

本文使用 UCI Machine Learning Repository 的 Bike Sharing Dataset。根据 UCI 数据集页面和本地 `Readme.txt`，该数据集记录美国华盛顿 D.C. Capital Bikeshare 系统 2011 至 2012 年按小时和按日汇总的共享单车租赁数量，并附加天气、季节、日期属性等变量。UCI 页面将其标记为多变量、回归任务数据集，数据规模为约 17.39K 个实例。

数据集必须引用：

```text
Fanaee-T, H. (2013). Bike Sharing [Dataset]. UCI Machine Learning Repository. https://doi.org/10.24432/C5W894
```

本地 README 同时要求引用 Fanaee-T 与 Gama 的论文：

```text
Fanaee-T, H., & Gama, J. (2013). Event labeling combining ensemble detectors and background knowledge. Progress in Artificial Intelligence. https://doi.org/10.1007/s13748-013-0040-3
```

该论文关注真实共享单车使用日志中的事件标注问题，说明共享单车数据不仅可用于需求预测，也可作为城市活动与异常事件检测的数据来源。对本文而言，原论文提供了两个重要背景：

1. 共享单车日志是现实交通系统产生的行为数据，不是人工构造数据。
2. 租赁量会受到季节、天气、日期和城市事件影响，因此适合用统计模型和预测模型分析。

## 2. 数据类型与变量逻辑

`hour.csv` 共有 17 个原始变量，可分为：

| 类型 | 变量 | 论文中的作用 |
|---|---|---|
| 索引与日期 | `instant`、`dteday` | 记录识别和时间范围说明 |
| 时间属性 | `yr`、`mnth`、`hr`、`weekday` | 年份、月份、小时和星期规律分析 |
| 日期类型 | `holiday`、`workingday` | 节假日和工作日差异分析 |
| 天气与环境 | `season`、`weathersit`、`temp`、`atemp`、`hum`、`windspeed` | 天气条件和骑行舒适度分析 |
| 租赁量 | `casual`、`registered`、`cnt` | 因变量和用户类型差异分析 |

关键业务关系：

```text
cnt = casual + registered
```

因此：

1. 建模 `cnt` 时不能把 `casual` 和 `registered` 作为自变量。
2. 分析用户结构时，可以分别以 `casual` 和 `registered` 作为因变量。
3. 相关系数中 `cnt` 与两类用户高度相关是定义关系，不是可直接解释的外生影响。

## 3. 数据处理逻辑

当前 R 脚本的数据处理包括：

1. 读取 `hour.csv` 和 `day.csv`。
2. 检查样本量、变量数、缺失值、重复值和 `cnt` 组成关系。
3. 将编码变量转换为中文因子标签。
4. 将标准化气温、湿度、风速还原为直观单位。
5. 构造 `log1p(cnt)`，用于线性回归缓解右偏分布影响。
6. 构造 `casual_ratio` 和 `registered_ratio`，用于用户结构分析。
7. 构造小时段变量，用于解释早高峰、晚高峰、夜间等需求结构。

这些处理步骤体现了统计论文中“先验证数据，再做分析”的原则。

## 4. 描述统计与可视化的理论作用

描述统计用于回答“数据是什么样”的问题。本文报告均值、中位数、标准差、四分位数、最小值和最大值，重点解释 `cnt` 的右偏分布。右偏分布说明少数高峰小时租赁量远高于普通小时，因此后续使用 `log1p(cnt)` 建模是合理的。

可视化用于发现结构性规律：

1. 小时折线图发现通勤高峰。
2. 工作日/非工作日对比图发现小时模式差异。
3. 天气箱线图发现天气状态与需求水平关系。
4. 温度平滑图发现连续环境变量的非线性趋势。
5. 相关系数图发现数据泄露边界和共线性风险。

这些图不是装饰，而是后续模型设定的依据。

## 5. 统计推断方法基础

本文使用 t 检验、Wilcoxon 秩和检验、ANOVA、Kruskal-Wallis 检验和 Tukey 事后比较。

使用逻辑如下：

| 方法 | 使用场景 | 解释重点 |
|---|---|---|
| t 检验 | 两组均值比较 | 工作日与非工作日、2011 与 2012 |
| Wilcoxon 秩和检验 | 两组非参数比较 | 在右偏分布下作为稳健补充 |
| ANOVA | 多组均值比较 | 季节、天气状况差异 |
| Kruskal-Wallis | 多组非参数比较 | 非正态分布下验证组间差异 |
| Tukey HSD | ANOVA 后事后比较 | 判断哪些组之间差异更明显 |

由于样本量较大，p 值容易显著。论文解释时不能只看 p 值，还应结合效应大小、分组均值和图形形态。

## 6. 线性回归与稳健诊断

本文使用 `log1p(cnt)` 作为线性回归因变量。这样做的原因是租赁量右偏且存在高峰值，直接建模原始 `cnt` 可能导致残差异方差更明显。

主模型形式为：

```text
log(1 + cnt) = f(season, yr, mnth, hr, holiday, workingday, weathersit, temp, hum, windspeed)
```

模型诊断包括：

1. VIF：检查多重共线性。
2. Breusch-Pagan 检验：检查异方差。
3. HC3 稳健标准误：在异方差存在时重新评估重点变量显著性。
4. 残差诊断图：查看残差模式、正态性和异常点。

Breusch-Pagan 检验来自 Breusch 和 Pagan（1979），用于检验线性回归误差方差是否随解释变量变化。稳健协方差估计的思想可追溯至 White（1980）。本文使用 `sandwich::vcovHC(type = "HC3")`，是因为真实观测数据中存在高峰时段和高杠杆点，HC3 较保守。

## 7. 计数模型基础

共享单车租赁量 `cnt` 是非负整数，因此 Poisson 回归是自然起点。Poisson 模型假设均值和方差相等，但真实租赁数据往往存在过度离散：高峰时段和好天气时需求波动明显更大。

当前脚本计算：

```text
Poisson 过度离散比 = Pearson 残差平方和 / 残差自由度
```

结果为 32.427，远大于 1，说明 Poisson 等均值方差假设不合适。因此进一步使用负二项回归。`MASS::glm.nb()` 通过估计额外离散参数 `theta`，允许方差大于均值，更适合过度离散计数数据。该方法依据 Venables 与 Ripley 的 MASS 体系。

## 8. LASSO 正则化模型基础

LASSO 通过在损失函数中加入 L1 惩罚，使部分系数收缩为 0，从而同时实现预测和变量选择。Friedman、Hastie 与 Tibshirani（2010）提出的 `glmnet` 使用循环坐标下降算法，沿正则化路径高效求解广义线性模型。

当前脚本使用：

```r
glmnet::cv.glmnet(x_train, y_train, alpha = 1, nfolds = 5)
```

其中：

1. `alpha = 1` 表示 LASSO。
2. `nfolds = 5` 表示五折交叉验证选择 `lambda`。
3. 预测时使用 `lambda.min`。
4. 自变量通过 `model.matrix()` 展开为哑变量矩阵。

论文中应解释：LASSO 的优势在于变量选择和防止过拟合，但如果真实关系高度非线性，线性正则化模型的预测性能可能弱于随机森林。

## 9. 随机森林与回归树基础

回归树通过递归二分将样本空间划分为若干区域，每个叶节点给出预测值。它直观、可解释，但单棵树容易受训练样本局部变化影响。

随机森林由 Breiman（2001）提出，通过对多棵树进行集成降低单棵树的不稳定性。每棵树基于随机样本和随机特征子集训练，最终通过平均得到回归预测。`ranger` 是 Wright 与 Ziegler（2017）实现的高效随机森林 R 包，支持分类、回归和生存森林。

当前脚本使用：

```r
ranger::ranger(
  predictor_formula,
  data = bike_train,
  num.trees = 500,
  mtry = 8,
  min.node.size = 10,
  importance = "permutation"
)
```

关键含义：

1. `num.trees = 500`：训练 500 棵树，降低集成方差。
2. `mtry = 8`：每次分裂候选变量数为 8。
3. `min.node.size = 10`：控制叶节点最小规模，避免过度拟合。
4. `importance = "permutation"`：通过置换变量衡量变量重要性。

本文随机森林测试集 RMSE 最低，说明共享单车需求中存在明显非线性和交互关系，如小时与工作日、温度与季节、天气与湿度之间的共同影响。

## 10. 模型评价指标

本文使用 RMSE、MAE 和 R 平方评价预测模型：

| 指标 | 含义 | 解释 |
|---|---|---|
| RMSE | 均方根误差 | 对大误差更敏感，适合关注高峰需求预测偏差 |
| MAE | 平均绝对误差 | 表示平均预测偏离量，更直观 |
| R 平方 | 测试集解释度 | 衡量模型对测试集变化的解释比例 |

评价必须在测试集上进行，不能只报告训练集结果。
