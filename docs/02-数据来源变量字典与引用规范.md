# 02 数据来源变量字典与引用规范

## 1. 本文使用的数据文件

本文使用 UCI Machine Learning Repository 中的 Bike Sharing Dataset。该数据集已经下载到本地：

```text
统计软件/
└── bike_sharing_dataset/
    ├── Readme.txt
    ├── day.csv
    └── hour.csv
```

论文主分析使用 `hour.csv`，补充分析可使用 `day.csv`。

选择 `hour.csv` 的理由：

1. `hour.csv` 是小时级数据，共 17,379 条记录，信息粒度更细。
2. 包含 `hr` 小时变量，可以分析早高峰、晚高峰、夜间低谷等出行规律。
3. 样本量足够支持回归模型、计数模型和机器学习预测模型。
4. 与 `day.csv` 相比，小时级数据更能体现 R 可视化和建模能力。

`day.csv` 的定位：

1. 用于日级趋势图或月度变化趋势的补充展示。
2. 用于帮助解释整体时间趋势。
3. 不作为主要建模数据，避免主线分散。

## 2. 数据背景

该数据集来自美国华盛顿 D.C. 的 Capital Bikeshare 系统，时间范围覆盖 2011 年和 2012 年。原始共享单车租赁记录经过汇总，形成小时级和日级租赁量数据，并加入了天气、季节、日期属性等外部信息。

论文中可以这样表述：

> 本文选取 UCI Machine Learning Repository 收录的 Bike Sharing Dataset 作为研究数据。该数据集以美国华盛顿 D.C. 的 Capital Bikeshare 共享单车系统为基础，记录了 2011 年至 2012 年期间按小时和按日汇总的单车租赁量，并包含季节、月份、星期、工作日、天气状况、温度、湿度和风速等变量。由于共享单车租赁需求通常受到时间规律和天气环境共同影响，该数据集适合用于开展综合性统计分析和预测建模。

注意：这段是原创转述，不要直接复制 UCI README 中的原句。

## 4. 变量字典

### 4.1 索引与时间变量

| 变量名 | 原始含义 | 论文中解释 | 处理方式 |
| --- | --- | --- | --- |
| `instant` | 记录编号 | 每条观测的顺序编号 | 不参与建模 |
| `dteday` | 日期 | 租赁记录对应的日期 | 转换为 Date 类型 |
| `yr` | 年份 | 0 表示 2011 年，1 表示 2012 年 | 转换为因子 |
| `mnth` | 月份 | 1 至 12 月 | 可作为因子或数值变量 |
| `hr` | 小时 | 0 至 23 时 | 作为因子进入模型 |
| `weekday` | 星期 | 一周中的第几天 | 转换为星期标签 |

### 4.2 日期属性变量

| 变量名 | 原始含义 | 论文中解释 | 处理方式 |
| --- | --- | --- | --- |
| `holiday` | 是否节假日 | 是否为节假日 | 因子变量 |
| `workingday` | 是否工作日 | 既非周末也非节假日时为工作日 | 因子变量 |

### 4.3 季节与天气变量

| 变量名 | 原始编码 | 论文中标签 |
| --- | --- | --- |
| `season` | 1 | 春季 |
| `season` | 2 | 夏季 |
| `season` | 3 | 秋季 |
| `season` | 4 | 冬季 |

天气状况变量 `weathersit`：

| 编码 | 论文中建议标签 | 含义概括 |
| --- | --- | --- |
| 1 | 晴朗或少云 | Clear, Few clouds, Partly cloudy |
| 2 | 多云或有雾 | Mist, Cloudy, Broken clouds |
| 3 | 小雨雪 | Light snow, Light rain, Thunderstorm |
| 4 | 恶劣天气 | Heavy rain, snow, fog 等 |

注意：`weathersit = 4` 在 `hour.csv` 中只有 3 条记录，建模解释时要谨慎。可以在描述统计中保留，但在某些模型或图表解释中说明样本数过少，不能过度解读。

### 4.4 环境连续变量

| 变量名 | 原始含义 | 标准化方式 | 论文解释 |
| --- | --- | --- | --- |
| `temp` | 标准化气温 | 除以 41 | 可还原为摄氏温度 |
| `atemp` | 标准化体感温度 | 除以 50 | 与气温高度相关，建模时谨慎同时使用 |
| `hum` | 标准化湿度 | 除以 100 | 可还原为百分比 |
| `windspeed` | 标准化风速 | 除以 67 | 可还原为实际风速 |

建议在论文解释中尽量使用还原后的单位：

```r
temp_celsius <- temp * 41
hum_percent <- hum * 100
windspeed_real <- windspeed * 67
```

这样写结果更容易理解。例如，与其写“temp 增加 0.1”，不如写“气温约增加 4.1 摄氏度”。

### 4.5 租赁量变量

| 变量名 | 含义 | 用途 |
| --- | --- | --- |
| `casual` | 非注册或临时用户租赁量 | 用户差异分析 |
| `registered` | 注册用户租赁量 | 用户差异分析 |
| `cnt` | 总租赁量 | 主要因变量 |

必须强调：

```text
cnt = casual + registered
```

因此，预测 `cnt` 时不能把 `casual` 和 `registered` 作为自变量，否则属于数据泄露。

## 7. R 中的变量转换建议

示例代码：

```r
bike_hour <- readr::read_csv("bike_sharing_dataset/hour.csv") |>
  dplyr::mutate(
    date = lubridate::ymd(dteday),
    season = factor(
      season,
      levels = c(1, 2, 3, 4),
      labels = c("春季", "夏季", "秋季", "冬季")
    ),
    yr = factor(
      yr,
      levels = c(0, 1),
      labels = c("2011年", "2012年")
    ),
    mnth = factor(mnth, levels = 1:12, labels = paste0(1:12, "月")),
    hr = factor(hr, levels = 0:23),
    holiday = factor(holiday, levels = c(0, 1), labels = c("非节假日", "节假日")),
    workingday = factor(workingday, levels = c(0, 1), labels = c("非工作日", "工作日")),
    weathersit = factor(
      weathersit,
      levels = c(1, 2, 3, 4),
      labels = c("晴朗或少云", "多云或有雾", "小雨雪", "恶劣天气")
    ),
    temp_celsius = temp * 41,
    hum_percent = hum * 100,
    windspeed_real = windspeed * 67,
    cnt_log = log1p(cnt),
    casual_ratio = casual / cnt,
    registered_ratio = registered / cnt
  )
```

## 8. 原创性与查重提醒

数据来源和变量说明可以参考 UCI 官方资料，但必须注意：

1. 不要复制 README 中关于共享单车背景的长段落。
2. 天气变量含义可以列表化翻译，但正文解释要用自己的话。
3. 变量表属于事实性说明，重复风险较低；但背景段落最容易重复，应简短。
4. 尽量把篇幅放在自己的 R 统计结果和图表解释上。
5. 数据引用要规范，引用规范本身不会提高重复率，反而能降低学术风险。
