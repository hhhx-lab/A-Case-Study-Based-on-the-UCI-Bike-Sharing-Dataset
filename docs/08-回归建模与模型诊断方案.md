# 08 回归建模与模型诊断方案

## 1. 本部分目标

回归建模是论文中“数据分析语言恰当性”和“内容丰富程度”的核心部分。探索性分析可以发现表面规律，但回归模型可以在控制其他变量后解释各因素与租赁量之间的关系。

本部分目标：

1. 建立多元线性回归模型解释共享单车租赁量。
2. 使用 `log1p(cnt)` 缓解右偏分布。
3. 加入小时、季节、天气、工作日和环境变量。
4. 检查多重共线性和残差诊断。
5. 使用稳健标准误增强结果可靠性。
6. 为后续计数模型和预测模型打基础。

## 2. 因变量选择

主要因变量：

```r
cnt_log = log1p(cnt)
```

选择原因：

1. `cnt` 是非负计数变量，且分布右偏。
2. 原始 `cnt` 的均值高于中位数，说明存在右尾。
3. 对数转换可以缓解异方差和极端高值影响。
4. `log1p()` 可以处理 0 值，虽然本数据中 `cnt` 最小值为 1。

论文写法：

> 由于每小时总租赁量 `cnt` 呈现右偏分布，本文在线性回归中使用 `log1p(cnt)` 作为因变量。该转换能够在保留观测顺序的同时降低极端高值对模型估计的影响，使回归结果更稳定。

## 3. 自变量选择

预测 `cnt` 的模型中使用：

| 类型 | 变量 |
| --- | --- |
| 时间变量 | `yr`, `mnth`, `hr`, `weekday` |
| 日期属性 | `holiday`, `workingday` |
| 季节天气 | `season`, `weathersit` |
| 环境变量 | `temp_celsius`, `hum_percent`, `windspeed_real` |

不使用：

| 变量 | 原因 |
| --- | --- |
| `casual` | `cnt` 的组成部分，数据泄露 |
| `registered` | `cnt` 的组成部分，数据泄露 |
| `instant` | 只是记录编号 |
| `dteday` | 日期原始变量，已通过年、月、星期等变量表达 |
| `atemp_celsius` | 与 `temp_celsius` 高度相关，避免共线性 |

## 4. 基准线性模型

R 代码：

```r
lm_base <- lm(
  cnt_log ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  data = bike_hour
)

summary(lm_base)
```

模型解释：

- `hr` 控制一天内不同小时的需求差异。
- `season` 和 `mnth` 控制季节和月份变化。
- `weathersit`、`temp_celsius`、`hum_percent`、`windspeed_real` 反映天气环境因素。
- `workingday` 和 `weekday` 控制工作日和星期差异。
- `yr` 控制年度趋势。

## 5. 交互项模型

共享单车需求可能存在交互效应。例如工作日和小时之间的关系：工作日早晚高峰更突出，非工作日白天更平缓。

推荐交互项：

```r
lm_interaction <- lm(
  cnt_log ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real +
    workingday:hr,
  data = bike_hour
)
```

可选交互项：

```r
temp_celsius:season
```

但不要加入太多交互项，否则模型表过长、解释困难。课程论文中推荐重点解释 `workingday:hr`。

论文写法：

> 鉴于探索性分析显示工作日与非工作日的小时租赁模式存在明显差异，本文进一步构造工作日与小时的交互项模型，以考察不同日期类型下小时效应是否不同。

## 6. 模型结果整理

使用 `broom`：

```r
lm_base_tidy <- broom::tidy(lm_base, conf.int = TRUE)
lm_interaction_tidy <- broom::tidy(lm_interaction, conf.int = TRUE)
lm_base_glance <- broom::glance(lm_base)
lm_interaction_glance <- broom::glance(lm_interaction)
```

使用 `modelsummary`：

```r
modelsummary::modelsummary(
  list(
    "基准模型" = lm_base,
    "交互项模型" = lm_interaction
  ),
  statistic = "({std.error})",
  stars = TRUE,
  output = file.path(table_dir, "table09_linear_models.html")
)
```

论文表格不必列出所有小时和月份虚拟变量，可以在正文中重点解释关键变量，把完整模型结果放附录。

## 7. 多重共线性检查

使用 `car::vif()`：

```r
vif_base <- car::vif(lm_base)
```

解释原则：

1. VIF 较高说明变量之间存在较强线性相关。
2. `temp_celsius` 和 `atemp_celsius` 不同时进入模型，就是为了减少共线性。
3. 分类变量多水平 VIF 解释需谨慎，可以报告广义 VIF。

论文写法：

> 为检验解释变量之间是否存在严重多重共线性，本文使用 VIF 指标进行诊断。考虑到气温和体感温度具有较强相关性，本文在主要模型中仅保留气温变量，以提高模型解释的稳定性。

## 8. 残差诊断

R 基础诊断图：

```r
png(file.path(fig_dir, "fig15_lm_diagnostics.png"), width = 2400, height = 1800, res = 300)
par(mfrow = c(2, 2))
plot(lm_base)
dev.off()
```

需要检查：

1. 残差与拟合值图：是否存在明显模式。
2. QQ 图：残差是否接近正态。
3. Scale-Location 图：是否存在异方差。
4. Cook 距离：是否存在强影响点。

论文写法：

> 残差诊断显示，线性模型能够捕捉主要变化趋势，但由于共享单车租赁量本身为计数型且存在高峰时段，残差仍可能存在一定异方差。因此，本文进一步使用稳健标准误和计数模型进行补充分析。

不要写“模型完全满足所有假设”。真实数据通常不会完美满足。

## 9. 异方差与稳健标准误

使用 `lmtest` 和 `sandwich`：

```r
robust_base <- lmtest::coeftest(
  lm_base,
  vcov = sandwich::vcovHC(lm_base, type = "HC3")
)
```

论文写法：

> 考虑到租赁量数据可能存在异方差，本文使用 HC3 稳健标准误对线性回归结果进行稳健性检验。若主要变量在稳健标准误下仍保持显著，则说明结论相对稳定。

## 10. 系数解释方法

因为因变量是 `log1p(cnt)`，系数解释不能直接说“增加多少辆”。建议：

对于连续变量：

> 在其他变量不变时，气温系数为正，说明气温升高与租赁量增加相关。

如果需要近似百分比解释：

```text
百分比变化约为 (exp(beta) - 1) * 100%
```

对于分类变量：

> 相对于基准组，某类别的系数为正，说明该类别下租赁量相对更高。

注意基准组取决于因子水平，例如：

- 春季可能是 `season` 基准组。
- 2011 年可能是 `yr` 基准组。
- 0 时可能是 `hr` 基准组。

## 11. 结果解释重点

正文重点解释以下变量：

1. 年份：2012 年是否显著高于 2011 年。
2. 小时：早晚高峰是否显著高于凌晨。
3. 季节：夏秋是否高于春季。
4. 天气：小雨雪是否显著低于晴朗天气。
5. 温度：是否正相关。
6. 湿度：是否负相关。
7. 风速：是否负相关。
8. 工作日与小时交互：工作日高峰是否更明显。

## 12. 回归章节写作模板

示例：

> 为进一步分析各因素对共享单车租赁量的影响，本文以 `log1p(cnt)` 为因变量建立多元线性回归模型。解释变量包括年份、月份、小时、星期、工作日、节假日、季节、天气状况、气温、湿度和风速。由于 `casual` 和 `registered` 是总租赁量的组成部分，本文未将二者作为解释变量，以避免数据泄露。

> 基准模型结果显示，在控制其他因素后，时间变量和天气环境变量仍与租赁量存在显著关系。小时效应表明租赁量在早晚通勤时段明显高于凌晨时段；天气变量表明天气状况较差时租赁量倾向于下降；环境变量中，气温与租赁量呈正相关，而湿度和风速与租赁量呈负相关。

## 13. 常见错误

1. 把 `casual` 和 `registered` 加进 `cnt` 模型。
2. 同时加入 `temp` 和 `atemp` 后不检查共线性。
3. 把对数模型系数解释成原始数量变化。
4. 只报告 R 平方，不解释变量方向。
5. 忽略残差诊断。
6. 认为显著变量就是因果变量。
7. 模型加入过多交互项导致无法解释。
