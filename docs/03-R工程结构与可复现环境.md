# 03 R 工程结构与可复现环境

## 1. 本部分目标

本部分用于指导后续如何组织 R 代码、输出图表、保存模型结果和整理附录。课程要求中明确提到“代码的简洁、清楚程度”和“使用工具的多少”，因此代码工程结构本身也是论文质量的一部分。

目标：

1. 建立清晰的项目目录。
2. 保证 R 脚本从原始 CSV 到图表、表格、模型结果能够一键复现。
3. 将所有输出统一保存，便于写论文时引用。
4. 保留 R 包版本和运行环境信息，作为附录的一部分。

## 3. R 环境策略

当前终端中 `Rscript` 暂时不在 PATH，这说明后续需要确认 R 或 RStudio 的安装方式。为了不混用 Python 环境，也不影响系统 PATH，R 只处理 R 相关分析，文档转换仍使用本机 Codex 文档工具链。

建议方案：

1. 如果本机已安装 RStudio，则直接在 RStudio 中打开 `统计软件` 目录作为项目目录运行脚本。
2. 如果命令行需要运行 R，则确认 `Rscript` 的实际路径，例如 `/usr/local/bin/Rscript` 或 `/opt/homebrew/bin/Rscript`。
3. 不使用 `sudo` 安装 R 包。
4. R 包安装到用户级 R library 或项目级 `renv` 环境。

推荐使用 `renv`，但如果课程时间紧，可以不强制。使用 `renv` 的好处是后续可复现性更强。

```r
install.packages("renv")
renv::init()
```

如果只是课程论文，最低要求是附录保留：

```r
sessionInfo()
```

## 6. 路径管理

推荐在脚本开头统一定义路径：

```r
data_dir <- "bike_sharing_dataset"
output_dir <- "outputs"
fig_dir <- file.path(output_dir, "figures")
table_dir <- file.path(output_dir, "tables")
model_dir <- file.path(output_dir, "models")

dir.create(output_dir, showWarnings = FALSE)
dir.create(fig_dir, recursive = TRUE, showWarnings = FALSE)
dir.create(table_dir, recursive = TRUE, showWarnings = FALSE)
dir.create(model_dir, recursive = TRUE, showWarnings = FALSE)
```

这样后续保存文件时路径统一，不会散落到工作目录。

## 10. 代码简洁原则

为了满足“代码简洁、清楚程度”评分要求，代码应遵守：

1. 不在脚本中重复读取同一数据。
2. 不把长路径反复写在每一行，统一定义路径变量。
3. 数据清洗集中在一个 `mutate()` 流程中完成。
4. 重复绘图可以封装简单函数，但不要过度抽象。
5. 每个输出文件有稳定命名。
6. 附录中的代码保留必要注释，删除调试代码。
7. 不把临时试错代码放入最终附录。

## 11. 可复现性检查清单

后续完成 R 脚本后，应从空目录状态检查：

1. 删除 `outputs/`。
2. 重新运行完整脚本。
3. 检查 `outputs/figures/` 是否重新生成所有图片。
4. 检查 `outputs/tables/` 是否重新生成所有表格。
5. 检查模型结果是否稳定。
6. 保存 `sessionInfo()`。

R 代码示例：

```r
sink(file.path(output_dir, "sessionInfo.txt"))
sessionInfo()
sink()
```
