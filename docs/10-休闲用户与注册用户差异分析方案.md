# 10 休闲用户与注册用户差异分析方案

## 1. 本部分目标

`casual` 和 `registered` 不能作为预测 `cnt` 的自变量，但它们非常适合作为独立分析对象。将用户拆分为休闲用户和注册用户，可以让论文从“总需求分析”进一步深入到“用户行为机制分析”，显著提升内容丰富程度。

本部分目标：

1. 比较休闲用户和注册用户的规模差异。
2. 分析两类用户在小时、工作日、季节和天气下的不同模式。
3. 分别建立模型解释两类用户租赁量。
4. 形成对共享单车通勤属性和休闲属性的讨论。

## 2. 基本事实

本地初步统计：

| 变量 | 均值 | 中位数 | 最大值 |
| --- | ---: | ---: | ---: |
| `casual` | 35.68 | 17 | 367 |
| `registered` | 153.79 | 115 | 886 |
| `cnt` | 189.46 | 142 | 977 |

可写结论：

1. 注册用户平均租赁量明显高于休闲用户。
2. 注册用户是总租赁需求的主要构成。
3. 休闲用户虽规模较小，但可能更能反映天气、周末和休闲出行影响。

## 3. 用户类型分析逻辑

本章建议围绕一个核心判断：

**注册用户更偏通勤属性，休闲用户更偏休闲和天气敏感属性。**

但这只是研究假设，必须通过图表和模型验证。

可检验的问题：

1. 注册用户是否在工作日早晚高峰更集中？
2. 休闲用户是否在非工作日白天更活跃？
3. 休闲用户是否更受天气和季节影响？
4. 两类用户对温度、湿度、风速的敏感程度是否不同？

## 4. 用户结构描述

### 4.1 用户占比

```r
user_share_summary <- bike_hour |>
  summarise(
    total_casual = sum(casual),
    total_registered = sum(registered),
    total_cnt = sum(cnt),
    casual_share = total_casual / total_cnt,
    registered_share = total_registered / total_cnt
  )
```

论文写法：

> 从累计租赁量看，注册用户贡献了共享单车系统的大部分使用量，说明该系统具有较强的稳定用户基础。休闲用户虽然占比较低，但其使用行为可能更容易受到天气、季节和休闲时间影响。

具体占比等 R 输出后补入。

## 5. 小时模式比较

### 5.1 代码

```r
user_hour_summary <- bike_hour |>
  group_by(hour_num) |>
  summarise(
    casual = mean(casual),
    registered = mean(registered),
    .groups = "drop"
  ) |>
  pivot_longer(
    cols = c(casual, registered),
    names_to = "user_type",
    values_to = "mean_rentals"
  ) |>
  mutate(
    user_type = recode(
      user_type,
      casual = "休闲用户",
      registered = "注册用户"
    )
  )

p_user_hour <- ggplot(
  user_hour_summary,
  aes(x = hour_num, y = mean_rentals, color = user_type)
) +
  geom_line(linewidth = 1) +
  geom_point(size = 1.8) +
  scale_x_continuous(breaks = 0:23) +
  labs(
    title = "休闲用户与注册用户的小时租赁模式",
    x = "小时",
    y = "平均租赁量",
    color = "用户类型"
  ) +
  theme_paper()
```

### 5.2 写作重点

如果图中注册用户在 8 时、17 至 18 时明显高峰，可写：

> 注册用户租赁量在早晚通勤时段明显上升，说明注册用户使用共享单车更可能与日常通勤相关。

如果休闲用户在中午至下午更高，可写：

> 休闲用户的租赁量曲线相对平缓，高峰更可能出现在白天或下午时段，说明休闲用户使用场景与通勤用户不同。

## 6. 工作日与非工作日比较

### 6.1 代码

```r
user_working_hour <- bike_hour |>
  group_by(workingday, hour_num) |>
  summarise(
    casual = mean(casual),
    registered = mean(registered),
    .groups = "drop"
  ) |>
  pivot_longer(
    cols = c(casual, registered),
    names_to = "user_type",
    values_to = "mean_rentals"
  ) |>
  mutate(
    user_type = recode(
      user_type,
      casual = "休闲用户",
      registered = "注册用户"
    )
  )

p_user_working <- ggplot(
  user_working_hour,
  aes(x = hour_num, y = mean_rentals, color = workingday)
) +
  geom_line(linewidth = 1) +
  facet_wrap(~ user_type, scales = "free_y") +
  scale_x_continuous(breaks = seq(0, 23, by = 2)) +
  labs(
    title = "不同日期类型下两类用户的小时租赁模式",
    x = "小时",
    y = "平均租赁量",
    color = "日期类型"
  ) +
  theme_paper()
```

### 6.2 写作重点

本图是用户差异章节最重要的图之一。它可以支持：

1. 注册用户在工作日有明显早晚高峰。
2. 休闲用户在非工作日可能更活跃。
3. 用户类型差异不是简单数量差异，而是使用场景差异。

- 如果注册用户在差天气下仍保持一定租赁量，可以说注册用户需求更稳定。

## 8. 用户类型回归模型

分别建立两个模型：

```r
lm_casual <- lm(
  log1p(casual) ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  data = bike_hour
)

lm_registered <- lm(
  log1p(registered) ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  data = bike_hour
)
```

模型比较输出：

```r
modelsummary::modelsummary(
  list(
    "休闲用户" = lm_casual,
    "注册用户" = lm_registered
  ),
  output = file.path(table_dir, "table14_user_models.html")
)
```

解释重点：

1. 比较 `workingday` 系数方向和显著性。
2. 比较天气变量影响。
3. 比较温度、湿度、风速影响。
4. 比较小时变量模式。

## 9. 用户占比模型可选

如果想进一步丰富，可以分析 `casual_ratio`：

```r
lm_casual_ratio <- lm(
  casual_ratio ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  data = bike_hour
)
```

但比例变量在 0 到 1 之间，普通线性模型只作为探索性补充即可，不建议作为主模型。

## 10. 本章写作模板

> 为进一步理解共享单车需求结构，本文将总租赁量拆分为休闲用户租赁量和注册用户租赁量。描述性统计结果显示，注册用户平均租赁量明显高于休闲用户，是总租赁需求的主要来源。但从行为模式看，两类用户并非仅存在数量差异，在小时分布、工作日差异和天气敏感性方面也可能存在不同。

> 小时曲线显示，注册用户在工作日早晚高峰时段租赁量明显上升，反映出较强的通勤属性；休闲用户的使用高峰则更可能集中在非工作日白天，说明其租赁行为更接近休闲出行。进一步的分组图和回归模型也表明，两类用户对日期属性和天气环境变量的响应存在差异。

具体结论需要根据真实图表和模型输出改写。

## 11. 运营含义

这一章可以自然引出运营建议：

1. 对注册用户，应重点保障工作日早晚高峰车辆供给。
2. 对休闲用户，应关注周末、节假日和好天气下的景区或休闲区域需求。
3. 在恶劣天气或高湿高风条件下，可适当调整调度预期。
4. 用户分群有助于更精细地进行车辆投放和运维安排。

## 12. 原创性与查重提醒

用户类型分析非常适合降低重复率，因为：

1. 它基于自己对 `casual` 和 `registered` 的拆分。
2. 图表和模型结果来自自己的 R 脚本。
3. 行为解释可以结合具体曲线，而不是复制通用背景。

注意不要写成营销报告式空话。每条结论都应先有图表或模型结果支撑。

## 13. 常见错误

1. 在 `cnt` 预测模型中错误加入 `casual` 和 `registered`。
2. 只比较均值，不分析时间模式。
3. 把注册用户直接等同于通勤用户，表述过于绝对。
4. 忽略休闲用户在周末和天气上的可能差异。
5. 用户类型模型没有与总量模型区分。
