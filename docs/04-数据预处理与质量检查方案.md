## 3. 缺失值检查

R 代码：

```r
missing_table <- bike_hour_raw |>
  summarise(across(everything(), ~ sum(is.na(.x)))) |>
  pivot_longer(
    cols = everything(),
    names_to = "variable",
    values_to = "missing_count"
  ) |>
  mutate(missing_rate = missing_count / nrow(bike_hour_raw))

readr::write_csv(missing_table, "outputs/tables/table03_missing_values.csv")
```

论文写法：

> 对小时级数据所有变量进行缺失值检查，结果显示各变量缺失值数量均为 0，说明该数据集完整性较好，后续分析无需进行缺失值插补。

注意：不要因为没有缺失值就不写。课程论文要体现“检查过”，而不是默认数据干净。

## 5. 业务一致性检查

必须检查：

```r
leakage_check <- bike_hour_raw |>
  mutate(check_cnt = casual + registered == cnt) |>
  count(check_cnt)
```

论文写法：

> 数据中 `casual` 表示休闲用户租赁量，`registered` 表示注册用户租赁量，二者之和等于总租赁量 `cnt`。本文通过 R 检查确认该关系成立。因此，在以 `cnt` 为因变量进行预测建模时，本文不将 `casual` 和 `registered` 作为解释变量，以避免数据泄露。

这是全文一个重要专业点，必须写清楚。

## 6. 分类变量转换

应将编码变量转换为具有实际含义的因子。

R 代码：

```r
bike_hour <- bike_hour_raw |>
  mutate(
    date = ymd(dteday),
    season = factor(
      season,
      levels = c(1, 2, 3, 4),
      labels = c("春季", "夏季", "秋季", "冬季")
    ),
    yr = factor(
      yr,
      levels = c(0, 1),
      labels = c("2011年", "2012年")
    ),
    mnth = factor(
      mnth,
      levels = 1:12,
      labels = paste0(1:12, "月")
    ),
    hr = factor(hr, levels = 0:23, labels = paste0(0:23, "时")),
    holiday = factor(
      holiday,
      levels = c(0, 1),
      labels = c("非节假日", "节假日")
    ),
    weekday = factor(
      weekday,
      levels = 0:6,
      labels = c("周日", "周一", "周二", "周三", "周四", "周五", "周六")
    ),
    workingday = factor(
      workingday,
      levels = c(0, 1),
      labels = c("非工作日", "工作日")
    ),
    weathersit = factor(
      weathersit,
      levels = c(1, 2, 3, 4),
      labels = c("晴朗或少云", "多云或有雾", "小雨雪", "恶劣天气")
    )
  )
```

注意：

1. 中文标签用于图表和论文解释。
2. 如果模型中使用中文因子导致输出不便，可以保留一份英文标签变量。
3. `hr` 作为因子可以捕捉非线性的小时效应，比直接用数值 0 至 23 更合理。

## 7. 连续变量还原

UCI 数据集中温度、湿度和风速为标准化变量。为便于解释，构造还原变量：

```r
bike_hour <- bike_hour |>
  mutate(
    temp_celsius = temp * 41,
    atemp_celsius = atemp * 50,
    hum_percent = hum * 100,
    windspeed_real = windspeed * 67
  )
```

论文写法：

> 原始数据中的气温、体感温度、湿度和风速均为标准化变量。为提高解释直观性，本文根据数据说明将其还原为摄氏度、百分比和实际风速指标，并在图表解释中优先使用还原后的变量。

## 8. 派生变量构造

### 8.1 小时段变量

```r
bike_hour <- bike_hour |>
  mutate(
    hour_num = as.integer(str_remove(as.character(hr), "时")),
    hour_group = case_when(
      hour_num >= 0 & hour_num <= 5 ~ "凌晨",
      hour_num >= 6 & hour_num <= 9 ~ "早高峰",
      hour_num >= 10 & hour_num <= 15 ~ "日间",
      hour_num >= 16 & hour_num <= 19 ~ "晚高峰",
      TRUE ~ "夜间"
    ),
    hour_group = factor(
      hour_group,
      levels = c("凌晨", "早高峰", "日间", "晚高峰", "夜间")
    )
  )
```

用途：

- 让图表解释更直观。
- 用于比较工作日和非工作日的出行规律。

### 8.2 对数因变量

```r
bike_hour <- bike_hour |>
  mutate(cnt_log = log1p(cnt))
```

解释：

`cnt` 是非负计数变量，分布右偏。`log1p(cnt)` 等于 `log(1 + cnt)`，可以缓解右偏，并避免 `cnt = 0` 时无法取对数的问题。虽然本数据 `cnt` 最小值为 1，但使用 `log1p()` 更稳健。

### 8.3 用户占比变量

```r
bike_hour <- bike_hour |>
  mutate(
    casual_ratio = casual / cnt,
    registered_ratio = registered / cnt
  )
```

用途：

- 分析不同时间和天气条件下休闲用户占比。
- 支持第八章用户类型差异分析。

## 9. 极端值与异常值处理

本论文不建议直接删除所谓异常值。共享单车租赁量的极端高值和极端低值可能是真实业务现象，例如早晚高峰、恶劣天气或特殊事件。

处理策略：

1. 描述统计中报告最小值、最大值和四分位数。
2. 用箱线图展示分布。
3. 回归模型中使用 `log1p(cnt)` 减少极端值影响。
4. 在讨论中说明极端值可能与特殊天气、节假日或城市事件有关，但本文不做外部事件因果识别。

论文写法：

> 本文未简单删除租赁量极端值，因为极端高峰和低谷本身可能反映共享单车需求的重要规律。为降低右偏分布对线性模型的影响，本文在回归分析中使用 `log1p(cnt)` 作为因变量进行稳健性比较。
