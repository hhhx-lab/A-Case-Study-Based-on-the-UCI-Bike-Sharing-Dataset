# 09 计数模型与机器学习预测方案

## 1. 本部分目标

线性回归适合解释变量方向和显著性，但共享单车租赁量 `cnt` 本质上是非负整数计数变量。因此，论文中进一步加入计数模型和机器学习预测模型，可以体现分析的全面性和 R 工具使用的丰富性。

本部分目标：

1. 使用 Poisson 回归处理计数型因变量。
2. 检查过度离散问题，并使用负二项回归作为改进。
3. 使用训练集和测试集比较模型预测效果。
4. 使用随机森林等机器学习模型提高预测表现。
5. 通过 RMSE、MAE、R 平方评价模型。
6. 区分“解释模型”和“预测模型”的作用。

## 2. 为什么需要计数模型

`cnt` 是每小时租赁总次数，具有以下特点：

1. 非负整数。
2. 分布右偏。
3. 存在高峰时段极端值。
4. 方差可能大于均值，即存在过度离散。

Poisson 回归是计数数据常用模型，但它假设均值和方差相等。真实共享单车数据通常波动更大，因此可能需要负二项回归。

论文写法：

> 由于总租赁量 `cnt` 是非负整数型变量，本文除线性回归外进一步使用 Poisson 回归进行计数建模。考虑到共享单车租赁量可能存在过度离散，即方差大于均值，本文同时建立负二项回归模型，并比较两类计数模型的拟合效果。

## 3. Poisson 回归

### 3.1 模型公式

```r
pois_model <- glm(
  cnt ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  family = poisson(link = "log"),
  data = bike_hour
)
```

### 3.2 结果整理

```r
pois_tidy <- broom::tidy(pois_model, conf.int = TRUE)
pois_glance <- broom::glance(pois_model)
```

### 3.3 过度离散检查

```r
overdispersion_ratio <- sum(residuals(pois_model, type = "pearson")^2) /
  pois_model$df.residual

overdispersion_ratio
```

解释：

- 如果该比值明显大于 1，说明存在过度离散。
- 如果过度离散明显，Poisson 回归的标准误可能偏小，显著性可能被高估。

论文写法：

> 本文通过 Pearson 残差平方和与残差自由度之比检验 Poisson 模型的过度离散情况。若该比值明显大于 1，则说明 Poisson 模型的等均值方差假设难以满足，有必要采用负二项回归作为补充。

## 4. 负二项回归

### 4.1 模型公式

```r
nb_model <- MASS::glm.nb(
  cnt ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  data = bike_hour
)
```

### 4.2 结果整理

```r
nb_tidy <- broom::tidy(nb_model, conf.int = TRUE)
nb_glance <- broom::glance(nb_model)
```

### 4.3 模型比较

```r
count_model_compare <- tibble(
  model = c("Poisson 回归", "负二项回归"),
  AIC = c(AIC(pois_model), AIC(nb_model)),
  BIC = c(BIC(pois_model), BIC(nb_model))
)
```

解释：

- AIC 和 BIC 越小，说明在复杂度惩罚下模型拟合相对更好。
- 如果负二项模型 AIC 明显低于 Poisson 模型，可以认为负二项模型更适合该数据。

## 5. 计数模型系数解释

Poisson 和负二项回归使用 log link，系数可以转化为发生率比：

```r
exp(coef(nb_model))
```

解释方式：

```text
IRR = exp(beta)
```

如果某变量 IRR 大于 1，说明该变量增加或该类别相对于基准组对应的预期租赁量更高；如果 IRR 小于 1，说明预期租赁量更低。

论文写法：

> 在计数模型中，系数经过指数化后可解释为发生率比。若某解释变量的发生率比大于 1，说明在其他变量不变时，该变量对应的预期租赁量较高；若发生率比小于 1，则说明对应的预期租赁量较低。

不要在正文中展开所有虚拟变量，重点解释方向稳定的变量。

## 6. 预测建模总体思路

解释模型重点是理解影响因素，预测模型重点是提高预测精度。论文中需要明确区分二者：

| 类型 | 目标 | 代表模型 |
| --- | --- | --- |
| 解释模型 | 理解变量方向和显著性 | 线性回归、负二项回归 |
| 预测模型 | 提高测试集预测准确度 | LASSO、随机森林、梯度提升 |

论文写法：

> 回归模型有助于解释各变量与租赁量之间的统计关系，但并不一定具有最优预测性能。因此，本文进一步采用训练集和测试集划分方法，比较传统回归模型与机器学习模型的预测效果。

## 7. 训练集与测试集划分

推荐使用 tidymodels：

```r
library(tidymodels)

set.seed(20260607)

bike_split <- initial_split(bike_hour, prop = 0.8, strata = cnt)
bike_train <- training(bike_split)
bike_test <- testing(bike_split)
```

注意：

1. 设置随机种子。
2. 可使用 `strata = cnt` 尽量保持目标变量分布。
3. 如果严格考虑时间序列预测，应按时间划分训练测试集。但课程论文中随机划分更容易展示模型比较。

可以在论文中说明：

> 本文采用 80% 训练集和 20% 测试集划分方式，并设置随机种子保证结果可复现。由于本文重点是比较不同模型对租赁量模式的拟合和预测能力，而非严格的未来时间序列预测，因此采用随机划分方式。

如果想更严谨，可以补充按年份划分：2011 训练，2012 测试。但这样预测难度更大。建议主文用 80/20，讨论中承认不是严格时间外推。


## 10. 随机森林模型

随机森林适合捕捉非线性和变量交互，是课程论文中很合适的机器学习补充。

```r
rf_recipe <- recipe(
  cnt ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  data = bike_train
) |>
  step_dummy(all_nominal_predictors())

rf_model <- rand_forest(
  trees = 500,
  mtry = tune(),
  min_n = tune()
) |>
  set_engine("ranger", importance = "permutation") |>
  set_mode("regression")

rf_workflow <- workflow() |>
  add_recipe(rf_recipe) |>
  add_model(rf_model)
```

为了简化，也可以先用固定参数：

```r
rf_model_simple <- rand_forest(
  trees = 500,
  mtry = 8,
  min_n = 10
) |>
  set_engine("ranger", importance = "permutation") |>
  set_mode("regression")
```

如果课程时间有限，固定参数随机森林已经足够展示工具使用。

## 11. LASSO 回归

LASSO 可作为正则化回归模型，用于变量选择和预测。

```r
lasso_recipe <- recipe(
  cnt ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit +
    temp_celsius + hum_percent + windspeed_real,
  data = bike_train
) |>
  step_dummy(all_nominal_predictors()) |>
  step_normalize(all_numeric_predictors())

lasso_model <- linear_reg(penalty = tune(), mixture = 1) |>
  set_engine("glmnet")
```

如果安装 `glmnet` 麻烦，可以把 LASSO 作为可选项，不强制。

## 12. 预测评价指标

必须报告：

| 指标 | 含义 | 方向 |
| --- | --- | --- |
| RMSE | 均方根误差，对大误差更敏感 | 越小越好 |
| MAE | 平均绝对误差，更直观 | 越小越好 |
| R 平方 | 预测解释度 | 越大越好 |

R 代码：

```r
prediction_metrics <- bind_rows(
  lm_pred |>
    metrics(truth = cnt, estimate = pred_lm) |>
    mutate(model = "线性回归"),
  rf_pred |>
    metrics(truth = cnt, estimate = pred_rf) |>
    mutate(model = "随机森林")
)
```

论文写法：

> 本文使用 RMSE、MAE 和 R 平方评价模型预测效果。其中 RMSE 对较大预测误差更敏感，MAE 表示平均绝对误差，R 平方反映模型对测试集租赁量变化的解释能力。

## 15. 常见错误

1. 预测模型中加入 `casual` 和 `registered`。
2. 只在训练集上评价模型。
3. 不设置随机种子。
4. 只报告一个模型，没有比较。
5. 只说随机森林更好，不解释为什么。
6. 把预测效果写成因果解释。
7. 不说明评价指标含义。
