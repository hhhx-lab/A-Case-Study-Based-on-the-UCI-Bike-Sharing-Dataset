# 《统计软件》期末论文写作与 R 分析计划

## 一、论文方向

论文题目暂定为：

**基于 R 语言的共享单车租赁需求影响因素与预测分析：以 UCI Bike Sharing 数据集为例**

本文以 UCI Machine Learning Repository 中的 Bike Sharing Dataset 为研究对象，使用 R 语言对 2011 年至 2012 年美国华盛顿 D.C. 共享单车小时级租赁数据进行系统分析。研究重点不是简单预测租赁量，而是从时间规律、季节特征、天气条件、工作日属性和用户类型差异等多个角度，全面分析共享单车租赁需求的变化规律，并比较传统统计模型与机器学习模型的预测效果。

核心分析目标：

1. 描述共享单车租赁数据的基本结构、变量含义和数据质量。
2. 分析租赁量在小时、星期、月份、季节等时间维度上的变化规律。
3. 分析天气、温度、湿度、风速等环境因素对租赁需求的影响。
4. 比较工作日、周末和节假日的租赁模式差异。
5. 分别分析休闲用户和注册用户的行为差异。
6. 建立并比较多种 R 模型，解释影响租赁量的主要因素，并进行预测评价。

数据来源：

- 数据集名称：Bike Sharing Dataset
- 数据来源：UCI Machine Learning Repository
- 原始作者：Hadi Fanaee-T
- 数据背景：Capital Bikeshare system, Washington D.C., USA
- 时间范围：2011 年至 2012 年
- 本地数据路径：
  - `bike_sharing_dataset/hour.csv`
  - `bike_sharing_dataset/day.csv`
  - `bike_sharing_dataset/Readme.txt`

主要使用数据：

- 主分析数据：`hour.csv`
- 样本量：17,379 条小时级记录
- 变量数：17 个
- 目标变量：`cnt`，表示该小时总租赁量

补充使用数据：

- 补充分析数据：`day.csv`
- 样本量：731 条日级记录
- 用途：用于展示日级趋势、月度趋势或与小时级结果相互印证

数据特点：

1. 数据量适中，既能支持统计建模，也不会过于复杂。
2. 没有缺失值，适合课程论文中进行完整的数据分析流程展示。
3. 变量类型丰富，包括时间变量、分类变量、连续环境变量和计数型因变量。
4. 适合使用 R 的数据整理、统计描述、可视化、推断检验和预测建模工具。

## 三、变量处理方案

### 1. 原始变量说明

| 变量名 | 含义 | 处理方式 |
| --- | --- | --- |
| `instant` | 记录编号 | 仅作为索引，不参与建模 |
| `dteday` | 日期 | 转换为 Date 类型 |
| `season` | 季节 | 转换为因子：春、夏、秋、冬 |
| `yr` | 年份 | 转换为因子：2011、2012 |
| `mnth` | 月份 | 转换为有序因子或数值变量 |
| `hr` | 小时 | 转换为因子，也可构造小时段 |
| `holiday` | 是否节假日 | 转换为因子 |
| `weekday` | 星期 | 转换为因子 |
| `workingday` | 是否工作日 | 转换为因子 |
| `weathersit` | 天气状况 | 转换为因子：晴、多云/雾、小雨雪、恶劣天气 |
| `temp` | 标准化温度 | 连续变量，也可还原为摄氏温度 |
| `atemp` | 标准化体感温度 | 连续变量，用于相关性比较 |
| `hum` | 标准化湿度 | 连续变量 |
| `windspeed` | 标准化风速 | 连续变量 |
| `casual` | 休闲用户租赁量 | 用于用户类型差异分析 |
| `registered` | 注册用户租赁量 | 用于用户类型差异分析 |
| `cnt` | 总租赁量 | 主要因变量 |

### 2. 派生变量

计划构造以下新变量：

| 新变量 | 含义 | 用途 |
| --- | --- | --- |
| `date` | 日期变量 | 时间序列趋势图 |
| `month_label` | 月份标签 | 月度分组图 |
| `hour_group` | 凌晨、早高峰、白天、晚高峰、夜间 | 解释通勤规律 |
| `weekend` | 是否周末 | 比较周末与工作日差异 |
| `cnt_log` | `log1p(cnt)` | 缓解右偏分布，用于线性回归 |
| `casual_ratio` | `casual / cnt` | 分析休闲用户占比 |
| `registered_ratio` | `registered / cnt` | 分析注册用户占比 |
| `temp_celsius` | `temp * 41` | 以实际摄氏温度解释结果 |
| `hum_percent` | `hum * 100` | 以百分比解释湿度 |
| `windspeed_real` | `windspeed * 67` | 以实际风速解释结果 |

### 3. 建模注意事项

预测 `cnt` 时，不能把 `casual` 和 `registered` 作为自变量放入模型，因为：

`cnt = casual + registered`

如果把两者作为自变量，会造成严重的数据泄露，使模型结果看似准确但没有实际意义。因此：

- `cnt` 模型只使用时间变量、日期属性、天气变量和环境变量。
- `casual` 和 `registered` 单独作为因变量，用于分析不同用户类型的行为差异。

4. 所有表格统一保存到 `outputs/tables/`。

### 模块 6：多元线性回归

目标：

- 使用传统统计模型解释租赁量影响因素。
- 以 `log1p(cnt)` 为主要因变量，缓解计数变量右偏问题。
- 解释小时、季节、天气、温度、湿度、风速等变量的影响方向和显著性。

基准模型：

```r
lm(
  log1p(cnt) ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit + temp + hum + windspeed,
  data = bike_hour
)
```

扩展模型：

```r
lm(
  log1p(cnt) ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit + temp + hum + windspeed +
    workingday:hr + temp:season,
  data = bike_hour
)
```

诊断内容：

- 残差分布图
- 拟合值与残差图
- QQ 图
- 多重共线性 VIF
- 稳健标准误

### 模块 7：计数回归模型

目标：

- 因为 `cnt` 是非负整数计数变量，进一步使用计数模型。
- 比较 Poisson 回归和负二项回归。

模型：

```r

  cnt ~ season + yr + mnth + hr + holiday +
    weekday + workingday + weathersit + temp + hum + windspeed,
  family = poisson(link = "log"),
  data = bike_hour
)
```

```r

  data = bike_hour
```

## 七、预期核心结论

在正式跑出结果前，论文可以围绕以下预期结论组织，但正文必须以 R 实际输出为准：

1. 共享单车租赁量具有显著的时间规律，早高峰和晚高峰较明显。
2. 工作日与非工作日的租赁模式不同，工作日更偏通勤，非工作日更偏休闲。
3. 季节和月份对租赁需求有明显影响，夏秋季通常高于春季。
4. 天气状况越差，租赁量越低。
5. 温度对租赁量有正向影响，湿度和风速可能有负向影响。
6. 注册用户数量远高于休闲用户，且注册用户的通勤规律更稳定。
7. 传统回归模型适合解释影响因素，随机森林等机器学习模型可能具有更好的预测表现。

| 论文结构合理性 | 按“数据-描述-探索-推断-建模-预测-结论”的逻辑展开 |

## 十、执行顺序

1. 创建 `scripts/analysis.R`，完成数据读取和预处理。
2. 输出描述统计表和数据质量检查表。
3. 生成所有探索性图表。
4. 完成统计检验并保存结果表。
5. 建立线性回归、计数回归和预测模型。
6. 生成模型比较表和变量重要性图。
7. 根据真实输出撰写论文正文。
8. 将 R 代码整理到附录。
9. 生成 Word 可编辑稿。
10. 由 Word 转 PDF，并检查 PDF 页数、图表、文字和公式格式。

## 十一、最终交付目标

最终应形成以下成果：

1. 一份结构完整的课程论文。
2. 一份可复现的 R 分析脚本。
3. 一批清晰的统计图表和模型结果表。
4. 一个可编辑 Word 版本。
5. 一个最终提交 PDF 版本。

本文应体现的核心能力是：不是简单套用一个模型，而是完整展示 R 在数据导入、清洗、统计描述、可视化、统计推断、回归建模、机器学习预测和论文结果输出中的综合能力。
